Dynamic Programming 动态规划

  • Algorithm

例子说明

例子来源：http://www.hawstein.com/posts/dp-novice-to-advanced.html

问题：现有面值为1元、3元和5元的硬币若干枚，如何用最少的硬币凑成11元？


解题思路：

首先该问题可以分解为较小的子问题，即如何用最少的硬币凑成 i 元？（i <= 11）

1、i=0, 如何用最少的硬币凑成0元。

显然用最少的硬币凑成0元需要0个硬币。这里用 d(i) = j 来表示凑够i元至少需要j个硬币，即d(0)=0。


2、i=1, 如何用最少的硬币凑成1元。

显然只能选面值为1的硬币，即d(1)=d(1-1)+1=d(0)+1=1。其中这里因为我选了1个1元的硬币，所以还剩下 1-1=0 元，所以只要加上凑成0元最少的硬币的数量即 d(0) 就行了。


3、i=2, 如何用最少的硬币凑成2元。

显然能选的硬币还是为面值为1的硬币，即选了一个面值为1的硬币后，还剩下 2-1=1 元，而这个又是上一个子问题d(1)=1，所以有d(2)=d(2-1)+1=d(1)+1=2。


4、i=3, 如何用最少的硬币凑成3元。

此时的选择是有两个的，即可以选择1元，也可以选择3元。

首先考虑选择面值为1元的情况，d(3)=d(3-1)+1=d(2)+1=3，即需要3个硬币。

若考虑选择面值为3元的情况，d(3)=d(3-3)+1=d(0)+1=1，即需要1个硬币。

综合两者，d(3)=min{d(3-1)+1, d(3-3)+1}=1。即最少需要1个硬币。


i = 5, 6, 7, 8, 9, 10, 11 的情况如上述所说以此类推。


从上面的分析中，有两个动态规划的重要概念，即状态和状态转移方程。

上述例子中的状态就是 d(i)，即凑够i元需要的最少硬币数量。

状态转移方程是譬如 d(3)=min{d(3-1)+1, d(3-3)+1}，抽象之后是 d(i)=min{ d(i-Vj)+1 }, 其中 i-Vj>0, Vj表示第j个硬币的面值。
